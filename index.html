目的

小学校低学年が楽しく算数を練習できるアプリ。ひらがな表示・選択式・絵ヒントで「わかった！」体験を設計した。

02
🏗️

二層構造の設計

子ども（操作者）と保護者（管理者）の二層構造。UIは子ども向けに徹しつつ、保護者ページで学習状況・設定を管理できる。

03
⚙️

技術選定

Flutter でクロスプラットフォーム対応。SharedPreferences でローカルデータを永続化。Python でビルドを自動化。

04
✅

7モード・2段階ヒント・保護者ページ・間違い履歴管理を実装。コードレビューで品質を高め、実運用レベルに仕上げた。

UI Design

2つの画面、
2人のユーザー

子ども画面はひらがなのみ・大きなタップターゲット・選択式で直感操作

保護者ページは演算別の正解率グラフ・間違い履歴・メニュー表示制御・難易度設定を一元管理

間違い履歴は永続保存。3回以上間違えた問題は🔴で強調、確認済みにして整理できる

メニューの表示/非表示を保護者が制御。学習段階に合わせてカスタマイズ可能

Development Journey

開発の流れ

📐

Phase 01 — 初期実装

7モードの基本実装

たし・ひき・かけ・わり算・文章問題・パズル・にがてこくふくを実装。SharedPreferences で設定と苦手リストを永続化。ひらがな統一でUI構築。

🔍

Phase 02 — コードレビュー

7つの問題を発見・修正

メモリリーク・重複バグ・0除算・setState内非同期・かけ算ハイライト誤りなどを自己レビューで発見。enum化・定数化でコード品質を向上。

👨‍👩‍👧

Phase 03 — 保護者ページ

二層UIの設計と実装

StatsManager クラスで演算別正解率を蓄積。設定・グラフ・苦手一覧・メニュー表示制御を統合した保護者管理画面を新設。

💡

Phase 04 — UX改善

ヒント2段階化・操作性向上

ヒントをテキスト→絵の2段階に。パズルにルール説明カード追加。手入力モードに「こたえあわせ！」ボタン。絵文字上限で表示崩れを防止。

🎊

Phase 05 — リザルト画面

達成感を演出するゴール体験

正解数・不正解数・正解率を表示するリザルト画面を実装。100%→🥇など正解率に応じたメダルとコメントでモチベーションを維持。

📋

Phase 06 — 履歴管理

消えない間違い履歴システム

HistoryManager クラスで問題ごとの間違い回数を永続記録。3回以上は🔴でハイライト。別画面で一覧表示し、確認済みフラグで整理できる設計。

Code Review

発見・修正した問題 × 7

Bug

① メモリリーク

TextEditingController を dispose していなかった。

@override void dispose() { _nameCtrl.dispose(); _ansCtrl.dispose(); super.dispose(); }
Bug

② 苦手リスト重複

同じ問題を間違えるたびに重複してリストに追加されていた。

// 追加前に重複チェック final exists = list.any((q) => q['m']==curM.name && q['n1']==n1); if (!exists) list.add({...});
Bug

③ パズル 0除算

slots[1]==0 のとき -1 を返し誤正解になるバグ。

if (slots[1] == 0) { setState(() => slots = List.filled(2,null)); return; // 自動リセット }
Refactor

④ enum でモード管理

文字列比較をタイプセーフな enum に置き換え。

enum MathMode { plus, minus, multi, div... bool get isPlus => this==plus || this==storyPlus; }
Bug

⑤ かけ算表ハイライト

OR条件で行・列が両方光っていた。AND条件で修正。

// 修正前（OR） final a=(r==n1||c==n2); // 修正後（AND） final a=(r==n1&&c==n2);
UX

⑥⑦ mounted チェック

setState 内処理を整理し mounted チェックを追加してクラッシュを防止。

setState(()=>wList=parsed); if(!mounted)return; if(wList.isEmpty){ Navigator.pop(context); }else{_gen();}
